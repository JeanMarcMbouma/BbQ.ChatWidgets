Interface IWidgetHintParser

Defines the contract for parsing widget hints from raw AI model output.

Namespace: BbQ.ChatWidgets.Abstractions
Assembly: BbQ.ChatWidgets.dll
Syntax
public interface IWidgetHintParser
Remarks

Widget hints are embedded in AI-generated text using XML-style markers (<widget>...</widget>). Implementations of this interface extract these hints, deserialize the widget definitions, and return both the cleaned content and the parsed widgets.

The parsing process:

  1. Identify widget markers in the raw output
  2. Extract JSON widget definitions between markers
  3. Deserialize JSON into typed ChatWidget instances
  4. Remove widget markers from content, leaving clean text
  5. Return tuple of (cleanContent, widgets)

Implementations should be resilient to malformed JSON and invalid widget types.

  • Widgets: A read-only list of successfully parsed widgets, or null if none found

The content is trimmed of excess whitespace.
Remarks

This method extracts widget definitions embedded in the format: <widget>{"type":"button","label":"Click","action":"submit"}</widget>

Multiple widgets can be present in a single output: "Here are options: <widget>{...}</widget> or <widget>{...}</widget>"

The method handles:

  • Multiple widget definitions in one output
  • Optional properties on widgets
  • Malformed JSON (skipped with warning/log)
  • Invalid widget types (skipped)
  • Empty widget markers (ignored)
  • No widgets present (returns null)
Examples

Input: "Click here: <widget>{"type":"button","label":"Submit","action":"submit"}</widget>" Output: ("Click here:", [ButtonWidget(label: "Submit", action: "submit")])
