    API Abstractions

    This section covers the interfaces you can implement to extend the system: IWidgetRegistry, IWidgetActionHandlerResolver, and IThreadService are good entry points.

    Core Interfaces

    Widget System

    • IWidgetRegistry - Central registry for widget templates
    • IWidgetToolsProvider - Provides widget tools to AI agents
    • IWidgetHintParser - Parses widget hints from AI responses
    • IWidgetHintSanitizer - Sanitizes widget content
    • IWidgetLocalizer - Localizes widget content
    • IRecyclableWidget - Interface for widgets that can be recycled

    Action Handling

    • IWidgetAction - Base interface for widget actions
    • IActionWidgetActionHandler<TWidgetAction, T> - Handler for typed widget actions
    • IWidgetActionRegistry - Registry for widget action metadata
    • IWidgetActionHandlerResolver - Resolves action handlers
    • IWidgetActionMetadata - Metadata for widget actions

    Thread Management

    • IThreadService - Manages conversation threads and history
    • IChatHistorySummarizer - Summarizes chat history to manage context windows

    AI Integration

    • IAIToolsProvider - Provides AI tools for the chat system
    • IAIInstructionProvider - Provides AI instructions and prompts

    SSE (Server-Sent Events)

    • IWidgetSseService - Real-time widget streaming via SSE
    • IStreamPayloadValidator - Validates SSE stream payloads
    • StreamValidationRules - Rules for stream validation

    Exceptions

    • PayloadValidationException - Exception for payload validation errors
    • PayloadValidationReason - Reasons for payload validation failures
    • PublishRateLimitExceededException - Exception for rate limit violations

    For a complete list of types, see the API Reference TOC.
