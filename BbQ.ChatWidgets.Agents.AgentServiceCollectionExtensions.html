    AddAgent<TAgent>(IServiceCollection, string, ServiceLifetime)

    Registers an agent type as a named keyed service in the dependency injection container.

    Declaration
    public static IServiceCollection AddAgent<TAgent>(this IServiceCollection services, string name, ServiceLifetime lifetime = ServiceLifetime.Scoped) where TAgent : class, IAgent
    Parameters
    Type Name Description
    IServiceCollection services

    The service collection.
    string name

    The unique name used to identify the agent in the registry.
    ServiceLifetime lifetime

    The service lifetime for the agent. Defaults to Scoped.
    Returns
    Type Description
    IServiceCollection

    The service collection for method chaining.
    Type Parameters
    Name Description
    TAgent

    The agent implementation type.
    Remarks

    Agents registered with this method can be retrieved via GetAgent(string) and are resolved directly from the DI container, respecting the specified lifetime.

    This method also ensures IAgentRegistry is registered (as scoped) if it has not already been added to the container.

    Usage:

    services.AddAgent<HelpAgent>("help-agent");
services.AddAgent<DataQueryAgent>("data-query-agent", ServiceLifetime.Singleton);
    Exceptions
    Type Condition
    ArgumentException

    Thrown when name is null or whitespace.

    The service collection.

    Optional global max-rounds configuration. Pass null to use defaults (MultiTurnAgentOrchestratorOptions).
    Returns
    Type Description

    The service collection for method chaining.
    Examples
    services.AddAgent<ResearchAgent>("research");
services.AddAgent<SummaryAgent>("summary");

        ("research", new AgentConversationOptions { Persona = "You are a research assistant." }),
        ("summary",  new AgentConversationOptions { Persona = "You are a concise summariser." })
    ],
    orchestratorOptions: new MultiTurnAgentOrchestratorOptions { MaxRoundsPerAgent = 3, MaxTotalRounds = 10 }

    AddMultiTurnAgentOrchestrator(IServiceCollection, string, IEnumerable<(string AgentName, AgentConversationOptions Options)>, MultiTurnAgentOrchestratorOptions?)

    Registers a MultiTurnAgentOrchestrator as a named keyed agent so that a TriageAgent<TCategory> (or any other routing agent) can delegate to it by name.

    Declaration
    public static IServiceCollection AddMultiTurnAgentOrchestrator(this IServiceCollection services, string name, IEnumerable<(string AgentName, AgentConversationOptions Options)> pipeline, MultiTurnAgentOrchestratorOptions? orchestratorOptions = null)
    Parameters
    Type Name Description
    IServiceCollection services

    The service collection.
    string name

    The unique name used to look up this orchestrator via GetAgent(string).
    IEnumerable<(string AgentName, AgentConversationOptions Options)> pipeline

    Ordered sequence of (agentName, options) pairs. Each agent must already be registered via AddAgent<TAgent>(IServiceCollection, string, ServiceLifetime).
    MultiTurnAgentOrchestratorOptions orchestratorOptions

    Optional global max-rounds configuration. Defaults to MultiTurnAgentOrchestratorOptions defaults if null.
    Returns
    Type Description
    IServiceCollection

    The service collection for method chaining.
    Examples
    // Register individual step agents
services.AddAgent<ResearchAgent>("researcher");
services.AddAgent<AnalystAgent>("analyst");

// Register the pipeline as a named agent the triage can route to
services.AddMultiTurnAgentOrchestrator(
    "data-pipeline",
    pipeline: [
        ("researcher", new AgentConversationOptions { Persona = "You are a research analyst." }),
        ("analyst",    new AgentConversationOptions { Persona = "You are a data analyst." })
    ]
);

// Route DataQuery intent to the multi-turn pipeline
Func<UserIntent, string?> routing = intent => intent switch {
    UserIntent.DataQuery => "data-pipeline",
    _                    => "help-agent"
};
