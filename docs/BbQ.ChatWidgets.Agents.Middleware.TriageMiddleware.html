    Remarks

    This middleware integrates the triage agent into the pipeline, ensuring that all requests are classified and routed through the appropriate agent.

    It can be used to:

    • Add a triage layer to existing pipelines
    • Ensure consistent request routing
    • Provide classification context to downstream middleware
    • Enable agent-to-agent communication through metadata

    Usage in pipeline:

    builder.Services.AddAgentMiddleware<TriageMiddleware>();

    Remarks

    Middleware implementations should:

    1. Perform pre-processing on the request if needed
    2. Call the next delegate to continue the pipeline
    3. Perform post-processing on the outcome if needed
    4. Return the outcome (modified or unmodified)

    The middleware can also choose not to call next, effectively short-circuiting the pipeline, but should return an appropriate Outcome.
